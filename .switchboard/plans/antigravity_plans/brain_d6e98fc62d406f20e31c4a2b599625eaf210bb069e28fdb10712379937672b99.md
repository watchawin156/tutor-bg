# Implementation Plan - Google Login, Online Storage, and UI/UX Revamp

This plan outlines the steps to migrate the TutorM application from a local-only, PIN-based system to a 100% online, Google Login-authenticated system, along with several UI/UX enhancements.

## User Review Required

> [!IMPORTANT]
> **Approval System**: I will implement a whitelist system. New users logging in via Google will see a "Pending Approval" screen. you will need to manually approve them by adding their email to the `users` table via Cloudflare Dashboard or a simple admin command.
>
> **Data Separation**: Data will be separated by Google Account (using a `user_id` field). When a user first logs in, their data will be empty unless we perform a one-time migration from their local storage.

## Verification Plan

### Automated Tests
- I will attempt to run `npm run lint` periodically to ensure no syntax errors.
- Since there are no existing integration tests, I'll rely on manual verification via browser for the Auth flow.

### Manual Verification
1. **Google Login**: Verify sign-in and sign-out flow.
2. **Approval Flow**: Sign in with an unapproved account and verify "Pending" status.
3. **Data Persistence**: Add a student, refresh the page, and verify the student is still there (fetched from D1).
4. **Mobile Layout**: Use Browser DevTools to verify:
    - Sidebar on right.
    - Financial cards in one row.
    - Date picker usability.
5. **PDF View**: Verify sidebar visibility while viewing reports.
